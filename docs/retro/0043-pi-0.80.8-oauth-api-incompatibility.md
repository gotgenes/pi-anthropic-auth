# Retro: #43 — Pi 0.80.8 OAuth API incompatibility

## Stage: Planning (2026-07-17T00:00:00Z)

### Session summary

Transformed the pre-existing analysis doc `docs/plans/0043-*.md` into the formal implementation plan for restoring `/login` on Pi 0.80.8+.
Confirmed the fix direction (Option A — drop the `oauth` override, delegate login/refresh to Pi's built-in `anthropicOAuth`) with the operator via two `ask_user` rounds, the second after explaining the consequences of losing `mergeRefreshedCredentials`.
The plan is a removal + dependency-floor raise + docs pass, so it dispatches to `/build-plan`, not `/tdd-plan`.

### Observations

- **Option A vs B was a genuine `ask_user` gate.** Even though the operator authored the issue recommending Option A, the trade-off (dropping `mergeRefreshedCredentials`) is breaking and fallback-semantics-shaped, so I surfaced it. The operator chose Option A, accepting the refresh-rotation risk; Option B (vendoring the flow) and an upstream follow-up were both explicitly declined and recorded as Non-Goals.
- **Consequence of losing `mergeRefreshedCredentials`, verified from source:** built-in `refreshAnthropicToken` (0.80.8) sets `refresh: data.refresh_token` verbatim; an omitted field becomes `undefined` and breaks the next refresh until manual re-login. Only bites if Anthropic ever omits a rotated token (RFC 6749 §6 permits it; observed behavior always rotates).
- **Breaking-change classification:** the peer-floor raise to `>=0.80.8` follows Issue #40's precedent (floor raise released as a major). The plan puts `fix!:` + `BREAKING CHANGE:` on the floor-raise commit (step 2), not the removal commit (step 1).
- **Step ordering matters for typecheck:** the module deletion must land before/with the devDep bump. Bumping to `0.80.8` while `src/anthropic-oauth.ts` still imports the removed functions from the now-types-only `/oauth` subpath would fail `tsc`. Step 1 deletes on `0.80.2` (functions still present); step 2 bumps and re-validates against `0.80.10`.
- **CHANGELOG is release-please-generated** — no manual `[Unreleased]` edit; the Conventional Commit subjects/footers drive it. Corrected the earlier issue-body assumption that a manual CHANGELOG entry was needed.
- **Operator-confirmed doc path fix folded in:** `~/development/pi-mono` references in `AGENTS.md` are wrong; the real clone is `~/development/pi/pi`. Added to the docs step (step 3) since `AGENTS.md` is already being edited.
- **Latest pi-ai is `0.80.10`** — chosen as the devDep pin target.
- Historical plan/retro records that mention the override (`0031`, `0033`, `0037`, `gap-analysis-and-next-steps.md`, retro `0027`) are left as-is (Non-Goals) — point-in-time records.
